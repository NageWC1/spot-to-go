 **SPECIFICATION**

**Overview**

The aim of this coursework is to write a concise dissertation proposal on the project that you have been allocated not exceeding 3,000 words (not including the bibliography).

**Description**

The dissertation proposal should include the following information.

*Background and description of the proposed project* – this should give a broad description of the project, provide some context into where this may be applied and state why it is important.

*Aim and objectives* \- the aim of the project and its associated objectives should be stated clearly, and the major tasks of the project identified.

*Critical review of related literature* – a broad range of related literature on theories/methods/techniques/tools should be reviewed, synthesised and analysed critically. Any gaps, challenges and potential methods should be identified.

*Methodologies and methods* \- considering information from the aims, objectives and literature review, a review of appropriate methodologies and/or methods should be given. This should state how the project will be undertaken and how its success will be evaluated. Diagrams and flowcharts may be useful to supplement the text. Any tools or evaluation methods should also be discussed.

*Analysis of risks and any ethical issues* – a risk analysis should be carried out to identify potential problems that may affect progress on the project. This may, for example, include obtaining any data or code externally, or requirements around specialised hardware or software. Further, if the project is likely to need ethical approval, this should be discussed and a plan given for how this will be obtained.

*Work plan and Gantt chart* – from the methodology design, a clear set of work packages should be identified. For each, a start date and duration should be stated along with a brief description of the work to be done, and any deliverables associated with this. A Gantt chart should be included to summarise the work plan.

*Structure and technical writing style* – the report should have a logical structure and be written in a technical style. References should be formatted correctly and need to be accurate and up-to-date.

**Deliverables**

The deliverable is a dissertation proposal report as specified above. The report can be written in either LaTeX or Word, using the appropriate template, but must be submitted as a PDF document. The total word count should not exceed 3,000 words (not including bibliography). Submission is through Blackboard. 

**Area focus**

1. Background and description of the proposed project (10%)  
2. Aim and objectives (10%)  
3. Critical review of related literature (20%)  
4. Methodologies and methods (20%)  
5. Analysis of risks and any ethical issues (10%)  
6. Project work plan and accompanying Gantt chart (20%)  
7. Structure and technical writing style (10%)
